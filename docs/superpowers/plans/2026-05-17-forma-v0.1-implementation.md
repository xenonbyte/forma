# Forma v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Forma v0.1 as a pnpm workspace with core storage/services, MCP tools, Fastify API, React Web admin, real Pencil integration, three-platform Agent installation, and verification scripts.

**Architecture:** Use a pnpm monorepo with `packages/core`, `packages/mcp`, `packages/server`, `packages/web`, `packages/agent`, and `packages/cli`. `core` owns all hard gates and filesystem state under `~/.forma`; MCP and Web API are thin adapters; Web UI reads through HTTP only; CLI packages and launches the system.

**Tech Stack:** Node 22+, pnpm 10.33, TypeScript, Vitest 4, Fastify, MCP TypeScript SDK, React, Vite, Tailwind CSS v4, LeaferJS, js-yaml, zod, Pencil CLI.

---

## Scope Check

The approved spec intentionally covers multiple subsystems. Keep the implementation testable by completing each task as a vertical slice with focused tests before moving on. The final implementation is still one v0.1 release because the user explicitly approved the complete v0.1 scope, three Agent platforms, real Pencil smoke, and design diff.

export type FormaErrorCode =
  | "PRODUCT_CONFIG_INCOMPLETE"
  | "REQUIREMENT_STATUS_INVALID"
  | "DOCUMENT_EMPTY"
  | "PAGES_EMPTY"
  | "PAGE_NOT_OWNED"
  | "PEN_FILE_INVALID"
  | "PAGE_NOT_DONE"
  | "NODE_NOT_FOUND"
  | "VERSION_TOO_LOW"
  | "HISTORY_FILE_MISSING"
  | "PRODUCT_NOT_FOUND"
  | "REQUIREMENT_NOT_FOUND"
  | "DESIGN_NOT_FOUND"
  | "STYLE_NOT_FOUND"
  | "PENCIL_CLI_NOT_FOUND"
  | "PENCIL_NOT_AUTHENTICATED"
  | "PENCIL_LOCK_HELD";

export class FormaError extends Error {
  constructor(
    public readonly code: FormaErrorCode,
    message: string,
    public readonly details: Record<string, unknown> = {}
  ) {
    super(message);
    this.name = "FormaError";
  }

  toJSON() {
    return { error_code: this.code, message: this.message, details: this.details };
  }
}
