# Forma v0.2 Design

Date: 2026-05-18
Source: `design-version/DESIGN-v2.md`
Status: Approved design for implementation planning

## Goal

Implement Forma v0.2 as one complete release batch:

- Add manual online style sync from `https://github.com/VoltAgent/awesome-design-md.git`.
- Generate updated style previews through the real Pencil CLI and the existing `_preview-template.pen`.
- Add a read-only baseline navigation graph to the Web admin.
- Keep default automated tests deterministic and offline, while adding a separate live test path for real GitHub and Pencil verification.

## Scope

v0.2 includes:

- `POST /api/styles/sync` to start a single asynchronous style sync task.
- `GET /api/styles/sync/status` to read persisted sync state and progress.
- A core `SyncService` that owns Git clone, style scanning, variable extraction, preview rendering, style index updates, cleanup, and crash recovery.
- Web style-library controls for starting sync, polling progress, rendering failed/running/idle states, and refreshing styles after completion.
- A baseline page Tab UI with the existing list view and a new visual navigation graph.
- A LeaferJS-backed `NavigationGraph` component and a separate deterministic force-layout module.
- Offline unit and integration coverage in `pnpm test`.
- A separate `pnpm test:live` path that uses real GitHub network access and real Pencil CLI/authentication.

v0.2 does not include:

- Scheduled automatic style sync.
- WebSocket progress streaming.
- Style sources other than `awesome-design-md`.
- Mutating existing product style selections during sync.
- Drag editing of navigation graph nodes.
- A new graph visualization dependency.
- Advanced graph layout libraries such as Dagre.
- Making live GitHub/Pencil tests part of default CI.

## Architecture

The implementation keeps the existing package boundaries.

| Package | v0.2 Responsibility |
| --- | --- |
| `packages/core` | Add `SyncService` and related pure helpers for scanning, variable extraction, hashing, categorization, status persistence, preview rendering orchestration, and crash recovery. |
| `packages/server` | Register sync API routes and map sync errors to stable HTTP responses. Routes delegate business logic to `core`. |
| `packages/web` | Add style sync controls and baseline navigation graph UI. Web code talks only to HTTP APIs and existing baseline data. |
| root scripts | Add `pnpm test:live` for real GitHub/Pencil validation without changing default `pnpm test` behavior. |

`SyncService` is the only module that coordinates style sync. Server routes do not read or write sync files directly. Web components do not infer filesystem state.

## Sync Service

`packages/core/src/sync.ts` exposes:

```typescript
export class SyncService {
  constructor(private deps: {
    home: string;
    pencilService: PencilService;
    runner: CommandRunner;
  }) {}

  async startSync(): Promise<{ task_id: string }>;
  async getStatus(): Promise<SyncStatus>;
  async recoverFromCrash(): Promise<void>;
}
```

The service persists state to `~/.forma/sync-state.yaml`.

State machine:

```text
idle -> running -> idle
              \-> failed -> running
```

`startSync()` performs entry gates before returning:

- `pencil version` succeeds.
- `pencil status` reports an authenticated active session.
- `git --version` succeeds within 5 seconds.
- No non-stale sync task is currently running.

If all gates pass, `startSync()` writes a `running` state with a generated `task_id`, starts an in-process async task, and returns immediately. The async task performs the full sync. It is not forked into another process because it must coordinate with the existing `PencilService` lock and is dominated by I/O.

## Sync Data Flow

The background task runs these phases:

1. `git_clone`
   - Clone `https://github.com/VoltAgent/awesome-design-md.git` with `--depth 1` into `/tmp/forma-sync-{task_id}/awesome-design-md`.
   - Use a 60 second timeout.
   - On timeout or clone failure, write `failed` state and keep existing local styles unchanged.

2. `scanning`
   - Scan first-level child directories only.
   - Include directories that contain `DESIGN.md`.
   - Skip directories whose names start with `_` or `.`.
   - If no style directories are found, fail with `Repository structure changed: no style directories found`.

3. `extracting_variables`
   - Read every `DESIGN.md`.
   - Extract style variables with deterministic regexes and fallback defaults.
   - Extract category by keyword rules from `DESIGN-v2.md`.
   - Extract description as the first non-empty, non-heading line, truncated to 50 characters.
   - Compute a SHA-256 hash of `DESIGN.md` for update detection.

4. `rendering_previews`
   - Copy `~/.forma/styles/_preview-template.pen` to the task temp directory for each style.
   - Use Pencil Agent Mode to set variables without modifying nodes.
   - Export `preview@2x.png` with `--export-scale 2`.
   - Validate that the exported PNG exists and has a valid PNG signature through existing Pencil export validation.
   - Move successful previews into `~/.forma/styles/{name}/preview@2x.png` with temp-file-plus-rename semantics.

5. `updating_index`
   - Copy each synced `DESIGN.md` into `~/.forma/styles/{name}/DESIGN.md`.
   - Atomically write `~/.forma/styles/styles.yaml`.
   - Include a top-level `last_synced` timestamp in `styles.yaml`.
   - Count `styles_added`, `styles_updated`, and `styles_failed`.
   - The public style metadata remains compatible with the existing `StyleMetadata` schema. Existing readers may ignore unknown top-level index metadata.

6. `cleanup`
   - Remove `/tmp/forma-sync-{task_id}` in `finally` on success and failure.

Single-style preview failure does not fail the whole sync. It increments `styles_failed`, keeps the old preview if one exists, and allows the Web UI to display the existing CSS placeholder if no preview is available.

## Pencil Locking

Preview rendering uses the existing `PencilService.withLock()`.

The sync task renders previews in batches of five styles:

```text
acquire lock -> render up to 5 previews -> release lock -> next batch
```

The batch boundary gives normal Agent design/refine operations a chance to run between sync batches. Because the existing lock implementation fails fast when another process holds the lock, `SyncService` should retry `PENCIL_LOCK_HELD` with a short delay before marking the current preview or batch as failed. The retry budget must be bounded so one blocked Pencil operation cannot stall sync forever.

## Crash Recovery

`createFormaStore()` creates `sync` alongside the existing services. Server startup triggers `store.sync.recoverFromCrash()`.

`buildServer()` is currently synchronous, so implementation must not depend on an awaited startup-only recovery call. `recoverFromCrash()` should also be applied by `startSync()` and `getStatus()` before they make decisions, so stale running state is corrected even if startup recovery is still pending or failed.

Recovery behavior:

- If `sync-state.yaml` is `running` and `started_at` is less than 10 minutes old, leave it as running.
- If it is `running` and older than 10 minutes, rewrite state to `failed` with a crash-recovery message.
- If it is `idle` or `failed`, do not change it.

Temporary task directories under `/tmp/forma-sync-*` are cleaned by the sync task itself. A later implementation may add a best-effort stale temp cleanup, but the v0.2 required recovery contract is the state reset.

## Web API

New routes:

- `POST /api/styles/sync`
- `GET /api/styles/sync/status`

`POST /api/styles/sync` returns:

- `202` with `{ task_id, status: "running", message }` when a task starts.
- `409` with `SYNC_ALREADY_RUNNING` when a non-stale task is already running.
- `503` with `PENCIL_CLI_NOT_FOUND`, `PENCIL_NOT_AUTHENTICATED`, or `SYNC_GIT_NOT_FOUND` for entry-gate failures.

`GET /api/styles/sync/status` returns the persisted `SyncStatus`:

- `idle` with optional `last_sync`.
- `running` with `task_id` and `progress`.
- `failed` with `task_id` and `error`.

Background task failures are not returned through the original POST response. They are persisted and exposed by the status endpoint.

## Web Style Library

`packages/web/src/api.ts` adds:

- `syncStyles()`
- `getSyncStatus()`

`StyleLibrary` expands its client prop to include sync methods. It keeps the existing search, category, variable filter, and grid/list controls.

UI behavior:

- On mount, fetch styles and sync status.
- If status is `running`, start polling every 2 seconds.
- When the user clicks sync, call `POST /api/styles/sync`, then poll status every 2 seconds.
- Stop polling when status becomes `idle` or `failed`.
- On successful completion, refresh the style list and show a compact summary: total, added, updated, failed.
- `idle` button label: `一键同步`.
- `running` button label: `同步中... ({current}/{total})`.
- `failed` button label: `同步失败，重试`.

The UI must not assume that a preview exists. Missing previews continue to use the existing style-card fallback behavior.

## Baseline Navigation Graph

`BaselineView` adds two tabs:

- `列表视图`: existing functional pages and navigation lists. This is the default tab.
- `导航图`: new graph view.

The graph uses only the existing `GET /api/products/:id/baseline` payload. No backend API or baseline schema change is required.

`packages/web/src/lib/force-layout.ts` provides a deterministic layout function:

```typescript
interface GraphNodeInput {
  id: string;
  name: string;
  features: string;
}

interface GraphEdgeInput {
  from: string;
  to: string;
  label?: string;
}
```

Output includes canvas size and node coordinates. The algorithm:

- Starts nodes in a circle around the canvas center.
- Uses fixed repulsion, attraction, damping, and 100 iterations.
- Centers a single node.
- Supports pages without navigation edges.
- Avoids new third-party layout dependencies.

`packages/web/src/components/NavigationGraph.tsx` renders with LeaferJS:

- Nodes are fixed-size rounded rectangles.
- Edges are directed lines with arrowheads.
- Edge labels use the existing API field `label`; the UI may describe this as the trigger text.
- Hovering a node highlights that node and related edges.
- Clicking a node shows a detail panel below the graph with full `features`, `copy`, `fields`, and `interactions`.
- Zoom and pan use Leafer runtime capabilities, following the same runtime-loader pattern as `AnnotationCanvas`.

Empty states:

- No pages: `暂无基线数据，请通过 Agent 上传需求`.
- Pages but no navigation: render nodes and show `暂无页面间导航关系`.
- One page: center one node.

## Error Codes

Add core error codes:

- `SYNC_ALREADY_RUNNING`
- `SYNC_GIT_NOT_FOUND`

Reuse existing error codes:

- `PENCIL_CLI_NOT_FOUND`
- `PENCIL_NOT_AUTHENTICATED`
- `PENCIL_LOCK_HELD`
- `PEN_FILE_INVALID`

HTTP mapping:

| Error code | HTTP status |
| --- | --- |
| `SYNC_ALREADY_RUNNING` | `409` |
| `SYNC_GIT_NOT_FOUND` | `503` |
| `PENCIL_*` | `503` except conflict-style lock handling inside background sync |
| validation/input errors | existing server mapping |

## Testing

Default `pnpm test` stays offline and deterministic.

Core tests:

- `scanStyles` includes first-level `DESIGN.md` directories and skips `_` / `.` names.
- `extractVariablesFromDesignMd` handles expected patterns and fallback defaults.
- category and description extraction follow the design rules.
- SHA-256 update detection counts added and updated styles.
- `startSync` rejects a non-stale running task.
- `recoverFromCrash` resets stale running state.
- sync status persistence reads and writes stable YAML.
- preview failure increments `styles_failed` and does not abort the whole sync when the runner is fake.

Server tests:

- POST starts sync and returns `202`.
- duplicate POST returns `409`.
- missing Git returns `503`.
- missing or unauthenticated Pencil returns `503`.
- status endpoint returns idle, running, and failed shapes.

Web tests:

- API client exposes sync methods and handles error payloads.
- `StyleLibrary` polls while running, disables the button, refreshes styles on completion, and shows failed state.
- `forceLayout` handles zero, one, disconnected, and connected graphs.
- `NavigationGraph` renders empty states and node detail selection with a mocked Leafer runtime.
- `BaselineView` switches between list and graph tabs without losing the existing list behavior.

Live tests:

- Add a separate `pnpm test:live` script.
- It requires real network access to GitHub, Git CLI, Pencil CLI, and authenticated Pencil status.
- It must clone the real `awesome-design-md` source and render at least one preview using `_preview-template.pen`.
- Failure should clearly name the missing dependency or failing external step.
- It must not be part of default `pnpm test` or normal CI unless a future CI job explicitly opts in.

## Verification

Implementation is complete when:

- `pnpm test` passes.
- `pnpm build` passes.
- `pnpm typecheck` passes.
- `POST /api/styles/sync` starts a sync task and returns `202`.
- `GET /api/styles/sync/status` reflects idle, running, and failed states.
- duplicate sync POST returns `409`.
- Git/Pencil entry-gate failures return `503`.
- successful sync updates `~/.forma/styles/styles.yaml`.
- successful sync writes `last_synced` in `~/.forma/styles/styles.yaml`.
- new synced styles have `DESIGN.md` and successful previews have `preview@2x.png`.
- single-style preview failures do not abort the whole sync.
- stale running state recovers to failed on server startup.
- the style library button and polling behavior match the status table.
- the baseline navigation graph displays nodes, directed edges, labels, hover state, click details, and empty states.
- `pnpm test:live` passes in an environment with network, Git, and authenticated Pencil.

## Implementation Notes

- Preserve existing public data shapes wherever possible.
- Do not migrate product configs that already reference older style metadata.
- Do not add new dependencies for graph rendering or layout.
- Use temp-file-plus-rename for persisted sync outputs.
- Keep live external dependency coverage separate from default local and CI checks.
- During implementation planning, verify version-sensitive external API details for React, LeaferJS, Fastify, and Vitest through Context7 before writing code that depends on them.
